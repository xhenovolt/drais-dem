# Quick Fix Reference - ESLint Common Patterns

## üöÄ INSTANT FIXES

### 1. Apostrophes in JSX Text
```tsx
// ‚ùå WRONG
<p>Don't worry</p>
<label>Teacher's comment</label>

// ‚úÖ CORRECT  
<p>Don&apos;t worry</p>
<label>Teacher&apos;s comment</label>
```

### 2. Quotes in JSX
```tsx
// ‚ùå WRONG
<div>He said "hello"</div>

// ‚úÖ CORRECT
<div>He said &ldquo;hello&rdquo;</div>
// OR
<div>He said {'"hello"'}</div>
```

### 3. Any Types in Error Handling
```typescript
// ‚ùå WRONG
catch (error: any) {
  console.error(error);
}

// ‚úÖ CORRECT
catch (error: unknown) {
  console.error(error instanceof Error ? error.message : 'Unknown error');
}
```

### 4. Any Types in Function Parameters
```typescript
// ‚ùå WRONG
function processData(data: any) {
  return data.id;
}

// ‚úÖ CORRECT
interface DataItem {
  id: number;
  name: string;
}
function processData(data: DataItem) {
  return data.id;
}
// OR if truly dynamic:
function processData(data: unknown) {
  if (typeof data === 'object' && data !== null && 'id' in data) {
    return (data as { id: number }).id;
  }
  throw new Error('Invalid data');
}
```

### 5. Any Types in Array/Object Operations
```typescript
// ‚ùå WRONG
const items: any[] = getItems();
const record: Record<string, any> = getData();

// ‚úÖ CORRECT
interface Item {
  id: number;
  name: string;
}
const items: Item[] = getItems();
const record: Record<string, string | number | boolean> = getData();
// OR
const record: Record<string, unknown> = getData();
```

### 6. Require Imports
```typescript
// ‚ùå WRONG
const module = require('./module');

// ‚úÖ CORRECT
import module from './module';
// OR dynamic import
const module = await import('./module');
```

### 7. Unused Variables
```typescript
// ‚ùå WRONG
const result = someFunction();
// result is never used

// ‚úÖ CORRECT - Option 1: Remove it
someFunction();

// ‚úÖ CORRECT - Option 2: Use it
const result = someFunction();
console.log(result);

// ‚úÖ CORRECT - Option 3: Prefix with underscore if intentionally unused
const _result = someFunction();
```

### 8. Prefer Const (when actually not reassigned)
```typescript
// ‚ùå WRONG
let name = 'John';
console.log(name); // Never reassigned

// ‚úÖ CORRECT
const name = 'John';
console.log(name);

// ‚ö†Ô∏è EXCEPTION - When actually reassigned (ESLint bug):
let { value } = obj;
value = value ?? default; // This IS reassignment - ignore ESLint
```

### 9. img to Image
```tsx
// ‚ùå WRONG
<img src="/photo.jpg" alt="Photo" width={100} height={100} />

// ‚úÖ CORRECT
import Image from 'next/image';

<Image src="/photo.jpg" alt="Photo" width={100} height={100} />
```

### 10. React Hooks Dependencies
```typescript
// ‚ùå WRONG
useEffect(() => {
  fetchData(userId);
}, []); // Missing userId dependency

// ‚úÖ CORRECT
useEffect(() => {
  fetchData(userId);
}, [userId]);

// ‚úÖ OR if intentional:
useEffect(() => {
  fetchData(userId);
// eslint-disable-next-line react-hooks/exhaustive-deps
}, []);
```

## üîß BULK FIXES

### Find and Replace All Apostrophes
**VS Code Regex Find:**
```
([A-Za-z])'([A-Za-z])
```

**Replace With:**
```
$1&apos;$2
```

**Example:**
- `Teacher's` ‚Üí `Teacher&apos;s`
- `Don't` ‚Üí `Don&apos;t`
- `Can't` ‚Üí `Can&apos;t`

---

### Find and Replace All "any" in Catch Blocks
**VS Code Regex Find:**
```
catch\s*\(\s*(\w+)\s*:\s*any\s*\)
```

**Replace With:**
```
catch ($1: unknown)
```

## ‚ö†Ô∏è COMMON PITFALLS

### 1. Don't Use Any for JSON
```typescript
// ‚ùå BAD
const data: any = JSON.parse(response);

// ‚úÖ GOOD
interface ResponseData {
  id: number;
  name: string;
}
const data: ResponseData = JSON.parse(response);
```

### 2. Don't Ignore Type Errors
```typescript
// ‚ùå BAD
// @ts-ignore
const result = dangerousOperation();

// ‚úÖ GOOD
// @ts-expect-error - Known limitation in library XYZ
const result = dangerousOperation();
```

### 3. Don't Over-Use Unknown
```typescript
// ‚ùå TOO VAGUE
function process(data: unknown) {
  // Have to check everything
}

// ‚úÖ BETTER
interface ProcessableData {
  id: number;
  type: string;
}
function process(data: ProcessableData) {
  // Type-safe!
}
```
